# CDK Redeem Only V0.2.1

本版本是 UPI 兑换稳定性修复版，重点修复失败 CDK 释放、Free 失败账号显示、自动续兑、日本节点注册兼容和第 7 步 2FA 重新认证。

## 修复

- 远端返回兑换失败、超时、拒绝、取消或未找到记录后，CDK 会重新计入可用池。
- `CDK 无效` 不会自动复用，避免坏卡反复匹配账号。
- Free 组账号只要兑换失败，会显示为兑换轮次并计入失败数量，不再误显示成待兑换。
- 失败 CDK 释放后，自动续兑会继续处理剩余待兑换账号。
- 日本节点下的注册入口、认证页切换和第 3 步慢响应做了兼容。
- 第 4/6 步验证码获取支持更稳的等待和自定义接口识别。
- 第 7 步开通 TOTP 2FA 遇到 `recent_auth_required` 时，会自动重新登录当前邮箱并刷新 accessToken/session/cookie 后重试。

## 使用说明

- 更新后在 Chrome 扩展页重新加载插件。
- 如 CDK 状态已经显示兑换失败，重新点击“一键兑换 CDK”即可继续用可恢复 CDK 匹配后续账号。
- 如果 CDK 显示为无效，请手动删除或停用该 CDK。
