# 项目文件结构说明

当前仓库为 UPI Redeem Only 专用版。

## 根目录

- `manifest.json`：Chrome 扩展清单，声明背景脚本、侧栏、内容脚本和权限。
- `background.js`：后台 service worker 主入口，装载 UPI 注册、邮箱、2FA、会员查询和卡密兑换模块。
- `README.md`：项目说明。
- `README-UPI-ONLY.md`：UPI 专用版说明。
- `Release.md`：发布说明。
- `docs/CONFIG-USAGE.md`：配置和使用说明。

## background

- `background/message-router.js`：处理侧栏消息和 UPI 操作消息。
- `background/auto-run-controller.js`：自动注册轮次、停止、重试和运行状态控制。
- `background/upi-credential-membership-checker.js`：Free/Plus 分组、AT 补充、Plus 识别/验证和卡密兑换。
- `background/verification-flow.js`：邮箱验证码轮询和提交。
- `background/steps/*.js`：主流程步骤实现。
- `background/*provider.js`：邮箱 provider 后台适配。

## content

- `content/signup-page.js`：OpenAI/Auth 页面自动化，负责注册、登录、密码、验证码、2FA、AT 读取和第六步错误页重试。
- `content/auth-page-recovery.js`：认证页恢复辅助。
- `content/*mail*.js`：邮箱页面取码辅助。
- `content/utils.js`、`content/operation-delay.js`：内容脚本公共工具。

## sidepanel

- `sidepanel/sidepanel.html`：侧栏页面结构。UPI-only 版本只展示账号、邮箱、密码、2FA、UPI、Free/Plus、卡密相关配置。
- `sidepanel/sidepanel.js`：侧栏交互、状态同步、按钮事件和导入导出。
- `sidepanel/*manager.js`：邮箱池、账号池、记录面板等 UI 管理器。

## data / shared / flows

- `data/step-definitions.js`：UPI 当前步骤定义。
- `data/address-sources.js`、`data/names.js`：资料填写用静态数据。
- `shared/source-registry.js`、`shared/flow-capabilities.js`、`shared/session-to-json-converter.js`：共享工具。
- `flows/openai/mail-rules.js`：邮箱验证码匹配规则。

## 已删除旧文件

UPI-only 版本不再包含旧支付流程、旧外部钱包支付、旧网络切换、手机验证取码、本地支付 helper 相关文件和侧栏入口。
